# GitLab Runner NixOS Role — Token Provisioning

Creates per-host GitLab Runner authentication tokens via the GitLab API and writes them to disk for the NixOS `services.gitlab-runner` service to consume.

## What This Role Does vs. The NixOS Flake

| Concern                                   | Handled By                                    |
| ----------------------------------------- | --------------------------------------------- |
| `services.gitlab-runner` NixOS config     | NixOS flake (generated by `inventory_builder`) |
| Docker virtualisation for runners         | NixOS flake                                   |
| Executor, tags, concurrent settings       | NixOS flake (from host_vars)                  |
| **Creating a runner in GitLab**           | **This role**                                 |
| **Writing the authentication token file** | **This role**                                 |
| **Verifying runner connectivity**         | **This role**                                 |

## Flow

1. Checks if `registrationConfigFile` already has a valid token (idempotent)
2. Verifies GitLab API is reachable
3. Creates runner via `POST /api/v4/user/runners` (GitLab 16+ endpoint)
4. Writes `CI_SERVER_TOKEN` to the token file on the target host
5. Restarts `gitlab-runner.service` to pick up the new token
6. Verifies the runner is registered in GitLab

## Requirements

- GitLab 16+ (uses new runner creation API)
- Admin-level API token in vault (`vault_secrets.gitlab.api_token`)
- NixOS host with `services.gitlab-runner` configured in its flake
- `gitlab_runner_enable: true` in host_vars or group_vars

## Usage

Called from `stage_6.yml` for hosts with `gitlab_runner_enable: true`:

```yaml
- hosts: 'nixos_vms:!gitlab_vms'
  roles:
    - role: gitlab_runner
      when: gitlab_runner_enable | default(false) | bool
```

Or run standalone:

```bash
cd ansible
ansible-playbook stage_6.yml --tags runners --limit docker-01
```

## Variables

See `defaults/main.yml` for all options. Key variables:

| Variable                             | Default                                | Description                                      |
| ------------------------------------ | -------------------------------------- | ------------------------------------------------ |
| `gitlab_runner_api_url`              | `https://git.{{ domain }}`             | GitLab instance URL                              |
| `gitlab_runner_api_token`            | from vault                             | Admin API token                                  |
| `gitlab_runner_token_file`           | `/var/lib/gitlab-runner/.runner-token` | Path matching `registrationConfigFile`           |
| `gitlab_runner_type`                 | `instance_type`                        | `instance_type`, `group_type`, or `project_type` |
| `gitlab_runner_tags`                 | `[docker, iac, {{ hostname }}]`    | Runner tags                                      |
| `gitlab_runner_skip_if_token_exists` | `true`                                 | Idempotent — skip if token file exists           |

## Tags

- `runners` — All runner tasks
- `token` — Token creation and deployment only
- `verify` — API and post-deploy verification
- `create` — Runner creation API call
- `info` — Display/summary tasks
